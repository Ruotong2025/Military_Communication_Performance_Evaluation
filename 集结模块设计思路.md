### 二、核心计算流程

```
① 选定 evaluation_id
   选定参与专家（全部 / 指定专家列表）
        ↓
② 计算每位专家的综合权重
   normCred_i = credibility.total_score / 100.0          // 0.01~1.0
   normConf_i = confidence_i（取该专家任一行的 confidence）
   rawWeight_i = normCred_i × 0.5 + normConf_i × 0.5    // 0.005~1.0
   normalizedWeight_i = rawWeight_i / ΣrawWeight_k       // 归一化，和为1
        ↓
③ 从 expert_ahp_comparison_score 取出所有比较打分
   按 comparison_key 分组，对每对比较：
     collective_score = Σ(normalizedWeight_i × score_ij)
        ↓
④ 构造 6 个集体判断矩阵
   ┌─ 维度层（5×5）
   ├─ 安全性指标层（3×3）
   ├─ 可靠性指标层（3×3）
   ├─ 传输能力指标层（6×6）
   ├─ 抗干扰能力指标层（3×3）
   └─ 效能影响指标层（3×3）
   
   填充规则：设 comparison_key = "A_B"，collective_score = s
   → A[A][B] = s,  A[B][A] = 1/s,  A[i][i] = 1
        ↓
⑤ 对 6 个矩阵分别做 AHP 计算（列和归一化→行均值→归一化）
   每个矩阵输出：权重向量 W、λmax、CR
   W_dim   (5个): 维度层权重
   W_sec   (3个): 安全性指标层权重
   W_rel   (3个): 可靠性指标层权重
   W_tra   (6个): 传输能力指标层权重
   W_aj    (3个): 抗干扰能力指标层权重
   W_eff   (3个): 效能影响指标层权重
        ↓
⑥ 计算 18 个二级指标综合权重（对总目标，归一化）
   W_combined["密钥泄露"] = W_dim["安全性"] × W_sec["密钥泄露"]
   W_combined["被侦察"]   = W_dim["安全性"] × W_sec["被侦察"]
   ...（18个指标同理）
   全部归一化：W_combined_k = W_combined_k / ΣW_combined
        ↓
⑦ 对 evaluation_id，从 score_military_comm_effect 获取所有 operation_id
   对每个 operation_id：
     指标加权得分_k = W_combined_k × indicator_score_k
     维度得分_j     = Σ(W_ind[dim][k] × indicator_score_k) / ΣW_ind[dim][k]
                        （维度内各指标加权得分的均值）
     综合得分       = Σ(W_dim_j × dimensionScore_j)
        ↓
⑧ 写入 expert_weighted_evaluation_result 表
   （每个 operation_id 一行，evaluation_id 不写入快照表）
```

### 三、专家综合权重计算

```java
// 公式
double normCred = credibility.totalScore / 100.0;   // 归一化到 [0,1]
double normConf = confidence;                       // 已是 [0,1]
double rawWeight = normCred * 0.5 + normConf * 0.5;
double normalizedWeight = rawWeight / ΣrawWeight;   // 归一化到 (0,1]，和为1
```

| 专家 | credibility | normCred | confidence | normConf | rawWeight | normalized |
|---|---|---|---|---|---|---|
| 专家1 | 88.5 | 0.885 | 0.85 | 0.85 | 0.8675 | 0.8675/2.4≈0.361 |
| 专家2 | 92.0 | 0.920 | 0.78 | 0.78 | 0.8500 | 0.8500/2.4≈0.354 |
| 专家3 | 85.0 | 0.850 | 0.10 | 0.10 | 0.4750 | 0.4750/2.4≈0.198 |

> 注：`confidence` 取该专家在 `expert_ahp_comparison_score` 中任一行的值（各行相同）。

### 四、集体比较打分（以「安全性_可靠性」为例）

```
专家1: score = 3,   normalized = 0.361
专家2: score = 5,   normalized = 0.354
专家3: score = 2,   normalized = 0.198

collective_score = 3×0.361 + 5×0.354 + 2×0.198 = 1.083 + 1.770 + 0.396 = 3.249
```

对全部 37 对比较执行此操作，得到 37 个 collective_score。

### 五、6 个集体矩阵的 AHP 计算

对每个矩阵（以维度层 5×5 为例）：

```java
// 1. 构造矩阵（设 collective_score(安全性,可靠性) = 3.249）
A[0][1] = 3.249;   A[1][0] = 1/3.249 = 0.3078;
其余元素同理，对角线为 1

// 2. 列和归一化 → 行均值 → 归一化
double[] W = expertAHPSvc.calculateWeights(matrix);

// 3. 计算 λmax、CR
double lambdaMax = expertAHPSvc.calculateLambdaMax(matrix, W);
double CR = (lambdaMax - n) / (n - 1) / RI[n];  // RI[5] = 1.12
```

对 6 个矩阵分别计算，得到 6 个 CR 值和 6 组权重。

### 六、综合权重计算

```
W_dim   = [安全性=0.201, 可靠性=0.198, 传输能力=0.215, 抗干扰=0.197, 效能影响=0.189]
W_sec   = [密钥泄露=0.35, 被侦察=0.33, 抗拦截=0.32]
W_rel   = [崩溃比例=0.34, 恢复能力=0.33, 通信可用=0.33]
W_tra   = [带宽=0.17, 呼叫建立=0.16, 传输时延=0.17, 误码率=0.17, 吞吐量=0.17, 频谱效率=0.16]
W_aj    = [信干噪比=0.34, 抗干扰余量=0.33, 通信距离=0.33]
W_eff   = [战损率=0.33, 任务完成率=0.34, 效费比=0.33]

综合权重（未归一化）：
  密钥泄露       = 0.201 × 0.35 = 0.07035
  被侦察         = 0.201 × 0.33 = 0.06633
  抗拦截         = 0.201 × 0.32 = 0.06432
  崩溃比例       = 0.198 × 0.34 = 0.06732
  ...（共18个）

归一化：W_combined_k = raw_k / Σ(raw_k)，使18个指标权重和为1
```

### 七、评估得分计算

```
设某 operation_id 的指标得分为 S_k：

指标加权得分_k = W_combined_k × S_k

维度得分（维度内加权平均）：
  dimensionScore_安全性 = Σ(W_sec[k] × S_k) / Σ(W_sec[k]) = 均值
  dimensionScore_可靠性 = Σ(W_rel[k] × S_k) / Σ(W_rel[k])
  ...（5个维度同理）

综合得分 = Σ(W_dim_j × dimensionScore_j)
```
