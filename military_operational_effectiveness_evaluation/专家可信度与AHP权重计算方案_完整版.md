# 军事通信效能评估 - 专家可信度与AHP权重计算方案

## 第一部分：理论基础 - 变异系数法在专家可信度评估中的应用

---

## 1 引言

### 1.1 研究背景

在军事通信效能评估领域，专家打分法是一种广泛应用的权重确定方法。然而，传统方法在处理专家评分数据时存在以下问题：

1. **离散度度量不准确**：常用相对离散度公式难以准确反映实际离散程度
2. **区分度不足**：难以设定统一的阈值进行一致性评判
3. **缺乏参考标准**：在军事领域的学术研究中缺乏合适的评判标准

### 1.2 研究目标

本研究旨在通过引入**变异系数（CV）**和**分段衰减函数**，建立一套科学、可量化的专家可信度评估体系，为军事通信效能指标权重计算提供可靠依据。

---

## 2 传统方法的局限性及改进方案

### 2.1 问题发现：传统相对离散度公式的局限性

传统相对离散度公式：

$$d = \frac{(x_{ij} - \bar{x}_j)^2}{x_{ij}^2 + \bar{x}_j^2}$$

**存在问题**：

1. 当专家评分普遍较低（数值小）时，即使评分差异很大，计算出的d值仍可能较小，导致**离散程度被低估**
2. 公式的**区分度不足**，难以设定统一的阈值进行一致性评判
3. 在军事领域的学术研究中**缺乏合适的参考标准**

### 2.2 改进方法：采用变异系数（CV）与分段衰减函数

为解决上述问题，本研究采用**变异系数（Coefficient of Variation, CV）**作为专家可信度的核心度量指标。

**CV的优势**：

- 消除量纲和均值大小的影响
- 能够更准确地反映评分的相对离散程度
- 具有明确的物理意义和统计理论基础

---

## 3 变异系数的核心概念与定义

### 3.1 两个维度的CV分析

本研究从两个维度进行考量：

| 维度 | 名称 | 含义 |
|------|------|------|
| 指标CV | 横向一致性 | 衡量所有专家对同一个指标评分的一致性。CV值越小，说明专家对该指标的看法越趋同。 |
| 专家CV | 纵向可信度 | 衡量某一个专家的评分与全体专家平均意见的偏离程度。CV值越小，说明该专家的评分风格越接近群体共识。 |

---

## 4 指标CV的计算与分析（跨专家一致性）

### 4.1 计算公式

$$\text{CV}_{\text{指标}} = \frac{\text{该指标所有专家评分的标准差}}{\text{该指标评分的平均值}} \times 100\%$$

> **注**：标准差计算采用样本标准差（ddof=1）。

### 4.2 计算示例：以"密钥泄露"指标为例

**专家评分**（把握度≥0.6）：

| 专家 | 张军 | 刘芳 | 赵敏 | 陈强 | 宋明 | 吴磊 | 刘雪梅 | 李辉 | 郑雪 | 李伟 |
|------|------|------|------|------|------|------|--------|------|------|------|
| 评分 | 6 | 5 | 7 | 5 | 7 | 6 | 7 | 6 | 6 | 5 |

**计算过程**：

1. **平均值** = (6+5+7+5+7+6+7+6+6+5)/10 = **6.00**
2. **标准差** = √[Σ(评分-均值)²/(n-1)] = **0.8165**
3. **CV** = (0.8165 / 6.00) × 100% = **13.61%**

---

## 5 专家CV的计算与分析（纵向可信度）

### 5.1 计算公式

$$\text{CV}_{\text{专家}} = \frac{\text{专家评分与各指标平均值的偏差的标准差}}{\text{整体评分平均值}} \times 100\%$$

其中：

- **整体平均值**：所有专家、所有指标评分的总平均
- **偏差**：某专家对某一指标的评分 — 该指标的全体专家平均分

### 5.2 计算示例：以"张军"专家为例

**偏差计算**：计算张军在每个指标上的评分与对应指标平均值的差值

| 指标 | 张军评分 | 指标均值 | 偏差 |
|------|----------|----------|------|
| 密钥泄露 | 6 | 6.00 | 0.00 |
| 被侦察 | 5 | 4.90 | +0.10 |
| ... | ... | ... | ... |

**计算过程**：

1. 偏差的平均值 = 0.1850
2. 偏差的标准差 = √(10.4055/19) = 0.7400
3. 专家CV = (0.7400 / 6.16) × 100% = **12.00%**

---

## 6 分段衰减函数

### 6.1 函数定义

结合军事领域对专家意见一致性的严格要求，并为了提升不同分歧程度间的区分度，设定如下**分段衰减函数**，用于将专家CV转换为客观可信度系数 β：

$$\beta_i = f(\text{CV}_i) = \begin{cases}
1.0 & \text{CV} \leq 15\% \\
1.0 - \frac{\text{CV} - 15}{10} \times 0.5 & 15\% < \text{CV} \leq 25\% \\
0.5 - \frac{\text{CV} - 25}{10} \times 0.5 & 25\% < \text{CV} \leq 35\% \\
0 & \text{CV} > 35\%
\end{cases}$$

### 6.2 阈值说明

| 区间 | 可信度等级 | 说明 |
|------|------------|------|
| CV ≤ 15% | 高度一致 | β = 1.0（完全可信） |
| 15% < CV ≤ 25% | 较为一致 | β 从 1.0 线性衰减至 0.5 |
| 25% < CV ≤ 35% | 轻度分歧 | β 从 0.5 线性衰减至 0.0 |
| CV > 35% | 严重分歧 | β = 0.0（失去意义，需重新打分） |

> **注**：该分段函数在 15%~25% 区间以较快速率（每增加1% CV，β下降0.05）衰减，在 25%~35% 区间保持相同速率继续衰减至零。

## 7 离散数据的处理与优化策略

### 7.1 问题识别

根据CV计算结果，识别需要处理的场景：

1. **专家CV > 35%**：该专家评分严重偏离群体共识，需要**重新打分**
2. **指标CV > 35%**：该指标专家分歧过大，需要**重新打分**或**删除离群值**

### 7.2 优化方案

#### 方案一：删除离群值（推荐用于个别极端值）

**触发条件**：当有效评分数量 ≥ 3 时

**操作步骤**：
1. 计算所有评分的均值
2. 计算每个评分与均值的偏差
3. 删除偏差最大的1条评分
4. 重新计算CV

**示例**：
- 功耗指标原始CV = 45%，删除最远值后CV = 28%
- 效费比指标原始CV = 42%，删除最远值后CV = 25%

#### 方案二：重新打分

当多个专家或多个指标的CV超过35%时，建议组织**新一轮专家咨询**，将评分向群体均值收敛。

### 7.3 优化效果验证

优化后应满足：

- 所有专家的CV < 35%
- 所有指标的CV < 35%
- 一致性显著改善

## 8 主观可信度评估（基于专家背景）

### 8.1 计算公式

$$\alpha_i = \gamma_1 \cdot \Omega_e + \gamma_2 \cdot \Omega_k$$

其中 **γ1 = γ2 = 0.5**（主观综合权重各占一半）

### 8.2 影响力得分 Ωe

$$\Omega_e = \sum_{m=1}^{6} \omega_m \cdot F_m$$

| 因素 | 权重 ωm | 评分字段 |
|------|---------|----------|
| 职称 | 0.20 | title_ql |
| 职务 | 0.15 | position_ql |
| 学历经历 | 0.15 | education_experience_ql |
| 学术成果 | 0.20 | academic_achievements_ql |
| 科研成果 | 0.20 | research_achievements_ql |
| 演习训练 | 0.10 | exercise_experience_ql |

### 8.3 专业知识得分 Ωk

$$\Omega_k = \sum_{m=1}^{4} \beta_m \cdot F_m$$

| 因素 | 权重 βm | 评分字段 |
|------|---------|----------|
| 军事训练学知识 | 0.30 | military_training_knowledge_ql |
| 系统仿真知识 | 0.25 | system_simulation_knowledge_ql |
| 数理统计学知识 | 0.20 | statistics_knowledge_ql |
| 专业年限 | 0.25 | professional_years_qt |

### 8.4 最终归一化

$$\alpha_i = \frac{\Omega_e + \Omega_k}{100}$$

---

## 9 综合可信度

### 9.1 计算公式

$$\text{综合可信度}_i = \lambda \cdot \alpha_i + (1-\lambda) \cdot \beta_i$$

其中 **λ = 0.6**（主观权重），**1-λ = 0.4**（客观权重）

### 9.2 最终结论

通过引入变异系数（CV）和分段衰减函数，我们能够：

1. **准确识别**需要重新打分的专家和指标
2. **量化可信度**：将CV转换为可用的可信度系数β
3. **优化数据**：通过删除离群值或重新打分提高数据质量
4. **提高可靠性**：优化后的评估结果稳定可靠，可用于后续的权重计算

## 12 AHP权重计算核心步骤

### 12.1 调整因子计算

对专家 i 在指标 j 的评分 wij，计算**调整因子** qij：

$$q_{ij} = 0.5 \times (0.6\alpha_i + 0.4\beta_i) + 0.5 \times \text{confidence}_{ij}$$

> 调整因子权重分配：
> - 把握度：0.5（直接反映该条评分的可靠程度）
> - 专家综合可信度：0.5（综合考虑专家主观和客观可信度）

### 12.2 把握度过滤

```
if confidence_ij < 0.6:
    该评分不参与后续任何计算
else:
    参与AHP计算
```

### 12.3 构建加权判断矩阵

对一级指标 k 下的二级指标集合 J = {j1, j2, ..., jn}：

**构造 n×n 判断矩阵 A：**

```
for i in J:
    for j in J:
        if i == j:
            A[i,j] = 1
        else:
            # 找出同时对两个指标都有把握(>=0.6)的专家
            有效专家集合 = { e | confidence_ei >= 0.6 AND confidence_ej >= 0.6 }

            if 有效专家集合为空:
                A[i,j] = 1
            else:
                # 每条评分乘以调整因子 q_e
                加权比值列表 = []
                for e in 有效专家集合:
                    加权评分_i = w_ei × q_ei
                    加权评分_j = w_ej × q_ej
                    比值 = 加权评分_i / 加权评分_j
                    加权比值列表.append(比值)

                # 【核心】离群值删除：当样本数≥3时，删除离均值最远的1个值
                if len(加权比值列表) >= 3:
                    均值 = mean(加权比值列表)
                    距离列表 = [abs(比值 - 均值) for 比值 in 加权比值列表]
                    删除索引 = 距离列表.index(max(距离列表))
                    del 加权比值列表[删除索引]

                # 几何平均
                A[i,j] = (∏比值)^(1/m)

            # 限制在1/9 ~ 9之间
            A[i,j] = max(1/9, min(9, A[i,j]))
```

### 12.4 一致性检验

$$\text{CI} = \frac{\lambda_{\text{max}} - n}{n - 1}, \quad \text{CR} = \frac{\text{CI}}{\text{RI}}$$

**RI值表：**

| n | RI |
|---|-----|
| 1 | 0 |
| 2 | 0 |
| 3 | 0.58 |
| 4 | 0.90 |
| 5 | 1.12 |
| 6 | 1.24 |
| 7 | 1.32 |
| 8 | 1.41 |
| 9 | 1.45 |
| 10 | 1.49 |

**判断标准**：CR ≤ 0.1 通过一致性检验

## 13 全员无把握指标处理

### 13.1 判定标准

如果某二级指标的所有专家评分都满足 confidenceij < 0.6，则该指标判定为**全员无把握**。

### 13.2 熵权法备用方案

当指标被判定为"全员无把握"时，使用**熵权法**基于 `equipment_operation_score` 表的数据来确定权重。

**计算步骤**：

1. **归一化处理**：
   - 效益型：rij = (xij - min(xj)) / (max(xj) - min(xj))
   - 成本型：rij = (max(xj) - xij) / (max(xj) - min(xj))

2. **计算信息熵**：
   $$E_j = -\frac{1}{\ln 10} \sum_{i=1}^{10} p_{ij} \ln p_{ij}$$

3. **计算权重**：
   $$w_j = \frac{1 - E_j}{\sum_{j=1}^{21}(1 - E_j)}$$

## 14 最终权重合成

### 14.1 合成公式

$$\text{最终权重}_j = \text{二级权重}_j \times \text{一级权重}_k$$

### 14.2 一级权重

| 一级指标 | 一级权重 |
|----------|---------|
| 安全性 | 0.25 |
| 可靠性 | 0.20 |
| 传输性能 | 0.20 |
| 抗干扰 | 0.15 |
| 资源消耗 | 0.10 |
| 作战效能 | 0.10 |

### 14.3 全局归一化

$$\text{最终权重}_{j,\text{归一化}} = \frac{\text{最终权重}_j}{\sum_{j=1}^{20} \text{最终权重}_j}$$

确保所有二级指标权重之和 **= 1**。
