## 一、方案概述

本方案在原有AHP权重计算和专家可信度评估的基础上，**引入惩罚模型**对军事通信装备效能进行综合评估。惩罚模型针对通信作战的底线要求，对核心指标设定阈值，确保低于阈值的指标会对最终效能得分产生惩罚性影响。

### 1.1 数据来源

| 数据表 | 说明 |
|--------|------|
| `ahp_final_weights` | AHP最终权重（20个二级指标） |
| `equipment_operation_score` | 10个实验的操作评分数据（21个评分维度） |

### 1.2 评估流程

```
AHP权重数据 (ahp_final_weights)
        ↓
实验评分数据 (equipment_operation_score)
        ↓
    计算阶段得分 Sstage = Σ(权重 × 指标得分)
        ↓
    应用惩罚模型 P = min(F_i)
        ↓
    最终得分 Sfinal = Sstage × P
        ↓
    生成柱状图可视化
```

## 二、惩罚模型设计（基于文献）

### 2.1 文献来源

> 王思远, 张目. 基于惩罚与激励的TOPSIS多属性决策方法[J]. 统计与决策, 2018, 34(10): 82-84.

### 2.2 核心惩罚指标

根据军事通信作战底线要求，选定以下**核心惩罚指标**：

| 指标名称 | 对应字段 | 数据来源 |
|----------|----------|----------|
| 密钥泄露得分 | security_key_leakage_qt | 军事作战效果指标 |
| 崩溃比例得分 | reliability_crash_rate_qt | 军事作战效果指标 |
| **连通率得分** | **system_performance_connectivity_rate_qt** | **装备操作层级** |
| 任务完成率得分 | effect_mission_completion_rate_qt | 军事作战效果指标 |

> **当前版本实现**：仅使用连通率作为核心惩罚指标

### 2.3 惩罚函数（文献公式）

设第 i 个核心指标得分为 x，定义惩罚因子 Fi：

$$F_i = \begin{cases} 1, & x \geq s \\ \frac{x}{s} \times m, & x < s \end{cases}$$

其中：
- **s** 为惩罚阈值，例如 **70分**
- **m** 为惩罚系数，**0 < m ≤ 1**，m 越小惩罚越重

### 2.4 综合惩罚因子

综合惩罚因子为各核心指标惩罚因子的**最小值**（取最严格的那个）：

$$P = \min(F_1, F_2, \cdots, F_n)$$

> 注：文献中采用"最小值"原则，即只要有一个核心指标低于阈值，就按该指标的惩罚因子计算，避免因多指标轻微惩罚而整体惩罚过轻。

### 2.5 最终效能得分

将全阶段综合得分乘以惩罚因子，得到最终效能得分：

$$S_{final} = S_{stage} \times P$$

## 三、惩罚模型参数配置

当前版本的惩罚模型配置如下：

```python
# 惩罚函数公式（文献）: F_i = 1, x ≥ s; F_i = (x/s)×m, x < s
penalty_config = {
    'system_performance_connectivity_rate_qt': {
        'name': '连通率',
        'threshold': 70,      # 惩罚阈值 s = 70分
        'm': 0.7,            # 惩罚系数 m = 0.8 (0<m≤1)
    }
}
```

### 3.1 惩罚效果示例

| 连通率得分 x | 惩罚因子 Fi = (x/s)×m | 说明 |
|--------------|----------------------|------|
| 100分 | 1.0000 | 无惩罚 |
| 90分 | 1.0000 | 无惩罚 |
| 80分 | 1.0000 | 无惩罚 |
| 70分 | 1.0000 | 刚好达到阈值 |
| 60分 | (60/70)×0.8 = 0.6857 | 惩罚约31% |
| 50分 | (50/70)×0.8 = 0.5714 | 惩罚约43% |
| 40分 | (40/70)×0.8 = 0.4571 | 惩罚约54% |

### 3.2 最终得分上限

由于惩罚因子的存在（m=0.8），**最终得分不会超过80%**：

- 当 x ≥ 70 时：Fi = 1，最终得分 = Sstage × 1 = Sstage（最高100%）
- 当 x < 70 时：Fi = (x/70)×0.8，最终得分被压缩到 80% 以下

## 四、效能计算实现

### 4.1 脚本文件

- **主脚本**: `calculate_effectiveness_with_penalty.py`

### 4.2 核心计算逻辑

```python
def calculate_penalty_factor(score_field, score_value):
    """
    计算惩罚因子（基于文献公式）
    
    公式: F_i = 1, x ≥ s; F_i = (x/s)×m, x < s
    """
    config = penalty_config[score_field]
    threshold = config['threshold']  # s: 惩罚阈值
    m = config['m']                  # m: 惩罚系数
    
    if score_value >= threshold:
        return 1.0
    else:
        # 惩罚因子 = (x/s) × m
        penalty = (score_value / threshold) * m
        return penalty

def calculate_effectiveness(self):
    """计算各实验的效能得分"""
    
    # 1. 获取AHP权重
    weights = self.get_ahp_weights()
    
    # 2. 获取实验评分
    experiment_scores = self.get_equipment_scores()
    
    # 3. 计算每个实验的效能得分
    for exp in experiment_scores:
        # 获取连通率（核心惩罚指标）
        connectivity_rate = float(exp.get('system_performance_connectivity_rate_qt', 0))
        
        # 计算连通率惩罚因子 Fi
        connectivity_penalty = self.calculate_penalty_factor(
            'system_performance_connectivity_rate_qt', 
            connectivity_rate
        )
        
        # 综合惩罚因子 P = min(Fi)（当前仅有一个指标）
        P = connectivity_penalty
        
        # 计算阶段得分 Sstage = Σ(权重 × 指标得分)
        stage_score = Σ(weight_info['final_weight'] * score_value)
        
        # 应用惩罚因子: Sfinal = Sstage × P
        final_score = stage_score * P
```

## 六、扩展说明

### 6.1 多指标惩罚

如需添加更多核心惩罚指标，可在 `penalty_config` 中添加配置：

```python
penalty_config = {
    'system_performance_connectivity_rate_qt': {
        'name': '连通率',
        'threshold': 70,
        'm': 0.8,
    },
    'reliability_crash_rate_qt': {
        'name': '崩溃率',
        'threshold': 60,  # 崩溃率阈值可能不同
        'm': 0.8,         # 惩罚系数
    },
    # 添加更多指标...
}
```

综合惩罚因子将自动更新为（取最小值）：

```python
P = min(F_connectivity, F_crash, ...)
```

### 6.2 惩罚系数 m 调整

可根据需要调整惩罚系数 m（0 < m ≤ 1）：

| m 值 | 惩罚效果 |
|-------|----------|
| m = 1.0 | 无惩罚（与原值相同） |
| m = 0.8 | 适度惩罚（当前） |
| m = 0.6 | 较强惩罚 |
| m = 0.4 | 严厉惩罚 |

### 6.3 阈值 s 调整

可根据实际作战要求调整惩罚阈值 s：

```python
# 更严格的阈值
'threshold': 80,  # 80分以上才不受惩罚

# 较宽松的阈值
'threshold': 60,  # 60分以上才不受惩罚
```

## 八、注意事项

1. **数据完整性**：确保 `ahp_final_weights` 和 `equipment_operation_score` 表中都有数据
2. **批次一致性**：AHP权重和实验评分应使用相同的批次ID
3. **字段匹配**：确保字段映射关系正确，否则指标得分将为0
4. **惩罚因子上限**：最终得分不会超过80%（当连通率<70分时，且m=0.8）
5. **文献一致性**：本方案严格遵循文献公式：
   - Fi = 1, x ≥ s
   - Fi = (x/s) × m, x < s
   - P = min(Fi)

## 十一、专家可信度评估方案

### 11.1 方案概述

专家可信度评估系统从**主观可信度**和**客观可信度**两个维度对专家进行综合评估。

### 11.2 主观可信度 (α)

基于专家背景信息计算：
- **影响力得分 (e)**：职称、职务、学历、工作年限等
- **知识得分 (k)**：专业领域、工作性质、评估经验等
- **主观综合得分 (i)**：=(e + k) / 2
- **主观可信度 (α)**：= i / max(i)

### 11.3 客观可信度 (β)

基于专家打分的离散程度计算：
- **向量相对离散度**：每位专家权重向量与平均向量的相对距离
- **CV（变异系数）**：专家打分与均值的偏差程度
- **客观可信度 (β)**：= 1 - 离散度（或基于CV计算）

### 11.4 综合可信度

```
综合可信度 = α × 主观权重 + β × 客观权重
```
