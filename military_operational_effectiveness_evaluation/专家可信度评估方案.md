# 专家可信度评估方案设计

## 一、数据源分析

### 1.1 expert_credibility_evaluation_score 表（主观评分数据）
包含10位专家的背景评分数据（0-100分制）：

**专家影响力因素（6项）：**
- `title_ql`: 职称得分
- `position_ql`: 职务得分
- `education_experience_ql`: 学历经历得分
- `academic_achievements_ql`: 学术成果得分
- `research_achievements_ql`: 科研成果得分
- `exercise_experience_ql`: 演习训练经历得分

**专业知识掌握程度因素（4项）：**
- `military_training_knowledge_ql`: 军事训练学知识得分
- `system_simulation_knowledge_ql`: 系统仿真知识得分
- `statistics_knowledge_ql`: 数理统计学知识得分
- `professional_years_qt`: 专业年限得分

### 1.2 ahp_expert_military_operation_effect_weights 表（客观权重数据）
包含10位专家对20个指标的权重评分（0-10分制）：
- 每位专家有20个权重值（对应20个军事作战效能指标）
- 每个权重还有对应的把握度（confidence，0-1）

## 二、计算方案设计

### 2.1 主观可信度 αi 计算

#### 步骤1：计算专家影响力得分 Ωe
```
Ωe = Ω1·Fzc + Ω2·Fzw + Ω3·Fxl + Ω4·Fxs + Ω5·Fky + Ω6·Fyl
```

**权重系数建议（Ω1-Ω6，总和=1）：**
- Ω1 (职称): 0.20 - 职称反映学术地位
- Ω2 (职务): 0.15 - 职务反映管理经验
- Ω3 (学历经历): 0.15 - 学历反映理论基础
- Ω4 (学术成果): 0.20 - 学术成果反映研究能力
- Ω5 (科研成果): 0.20 - 科研成果反映实践能力
- Ω6 (演习训练): 0.10 - 演习经历反映实战经验

#### 步骤2：计算专业知识掌握程度得分 Ωk
```
Ωk = β1·Fzx + β2·Fxf + β3·Fst + β4·Fsj
```

**权重系数建议（β1-β4，总和=1）：**
- β1 (军事训练学): 0.30 - 军事知识最重要
- β2 (系统仿真): 0.25 - 仿真能力很重要
- β3 (数理统计): 0.20 - 统计分析能力
- β4 (专业年限): 0.25 - 经验积累

#### 步骤3：计算主观综合得分 Ωi
```
Ωi = γ1·Ωe + γ2·Ωk
```

**权重系数建议（γ1, γ2，总和=1）：**
- γ1 (影响力): 0.5
- γ2 (专业知识): 0.5

#### 步骤4：标准化得到主观可信度 αi
```
αi = Ωi / Σ(Ωj)  (j=1 to m)
```

### 2.2 客观可信度 βi 计算

#### 步骤1：构建权重向量
从 `ahp_expert_military_operation_effect_weights` 表提取每位专家的20个权重值：
```
wi = (w1i, w2i, ..., w20i)
```

**20个指标的权重字段：**
1. security_key_leakage_weight
2. security_detected_probability_weight
3. security_interception_resistance_weight
4. reliability_crash_rate_weight
5. reliability_recovery_capability_weight
6. reliability_communication_availability_weight
7. transmission_bandwidth_weight
8. transmission_call_setup_time_weight
9. transmission_transmission_delay_weight
10. transmission_bit_error_rate_weight
11. transmission_throughput_weight
12. transmission_spectral_efficiency_weight
13. anti_jamming_sinr_weight
14. anti_jamming_anti_jamming_margin_weight
15. anti_jamming_communication_distance_weight
16. resource_power_consumption_weight
17. resource_manpower_requirement_weight
18. effect_damage_rate_weight
19. effect_mission_completion_rate_weight
20. effect_cost_effectiveness_weight

#### 步骤2：计算平均权重向量
```
w̄ = (w̄1, w̄2, ..., w̄20)
w̄j = (1/m) · Σ(wji)  (i=1 to m)
```

#### 步骤3：计算贴近度 ηi

**原始公式（论文中）：**
```
ηi = 1 - √[Σ(wji - w̄j)²] / √[Σ(wji)² + Σ(w̄j)²]
```

**改进方案：使用余弦相似度**
```
ηi = Σ(wji · w̄j) / (√[Σ(wji)²] · √[Σ(w̄j)²])
```

**为什么要改进？**

1. **数学意义更清晰**
   - 原公式：基于欧氏距离的归一化，分母包含两个向量的模，但形式不够标准
   - 余弦相似度：直接度量两个向量的夹角，是标准的相似度度量方法

2. **计算稳定性更好**
   - 原公式：当权重向量模很大时，分母可能导致数值不稳定
   - 余弦相似度：归一化后的结果在 [0, 1] 或 [-1, 1] 范围内，数值稳定

3. **解释性更强**
   - 原公式：1减去某个距离比值，不够直观
   - 余弦相似度：值越接近1表示方向越一致，越接近0表示方向差异大

4. **广泛应用**
   - 余弦相似度在推荐系统、文本相似度、专家一致性分析中被广泛使用
   - 有成熟的理论基础和实践经验

**结论**：使用余弦相似度作为贴近度度量，范围 [0, 1]，值越大表示专家权重与群体平均越接近

#### 步骤4：标准化得到客观可信度 βi
```
βi = ηi / Σ(ηj)  (j=1 to m)
```

---

### 2.3 综合可信度计算

#### 方案A：加权平均
```
Credibility_i = λ·αi + (1-λ)·βi
```
建议 λ = 0.6（主观权重稍高）

#### 方案B：乘积形式
```
Credibility_i = (αi · βi) / Σ(αj · βj)
```

## 三、实现步骤

### 3.1 数据准备
1. 从 `expert_credibility_evaluation_score` 表读取10位专家的背景数据
2. 从 `ahp_expert_military_operation_effect_weights` 表读取10位专家的权重数据

### 3.2 主观可信度计算
1. 计算每位专家的影响力得分 Ωe
2. 计算每位专家的专业知识得分 Ωk
3. 计算主观综合得分 Ωi
4. 标准化得到 αi

### 3.3 客观可信度计算
1. 提取每位专家的20维权重向量
2. 计算群体平均权重向量
3. 计算每位专家与平均向量的贴近度（余弦相似度）
4. 标准化得到 βi

### 3.4 综合可信度计算
1. 使用加权平均或乘积形式计算综合可信度
2. 输出结果并保存到新表
