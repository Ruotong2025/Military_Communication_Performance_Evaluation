# 无线通信关键指标的公开标准数据来源

## 文档目的
本文档提供 **平均丢包率**、**有效吞吐量**、**频谱效率** 三个指标的公开标准数据来源和评估范围，用于军事无线通信系统效能打分。

---

## 一、平均丢包率（EF_avg_plr - Packet Loss Ratio）

### 1.1 ITU-T Y.1541 标准（主要参考）

**标准名称**：ITU-T Y.1541 "Network performance objectives for IP-based services"

**官方下载**：https://www.itu.int/rec/T-REC-Y.1541/

**标准内容**：定义了IP网络的丢包率（PLR）性能目标

#### QoS 等级分类

| QoS等级 | 应用类型 | 丢包率（PLR）上限 | 适用场景 |
|---------|---------|-----------------|---------|
| **Class 0** | 实时、抖动敏感、高交互 | **10^-3 (0.1%)** | VoIP、视频会议 |
| **Class 1** | 实时、抖动敏感、交互 | **10^-3 (0.1%)** | VoIP、实时视频 |
| **Class 2** | 事务数据、高交互 | **10^-3 (0.1%)** | 信令、交易数据 |
| **Class 3** | 事务数据、交互 | **10^-3 (0.1%)** | 信令、命令控制 |
| **Class 4** | 低丢失、短事务 | **10^-3 (0.1%)** | 短消息、查询 |
| **Class 5** | 无约束 | **10^-2 (1%)** | 传统应用 |
| **Class 6** | 视频流、音频流 | **10^-3 (0.1%)** | 流媒体 |
| **Class 7** | 广播视频 | **10^-5 (0.001%)** | 广播电视 |

**关键引用**：
> "For Class 0-4 and 6, the packet loss ratio should not exceed 10^-3"
> "For Class 5, the packet loss ratio should not exceed 10^-2"

**查询方法**：
1. 访问 ITU 官网：https://www.itu.int/
2. 搜索 "Y.1541"
3. 免费下载PDF版本

---

### 1.2 ITU-T G.1010 标准（补充参考）

**标准名称**：ITU-T G.1010 "End-user multimedia QoS categories"

**官方下载**：https://www.itu.int/rec/T-REC-G.1010/

**标准内容**：定义了端到端多媒体服务的QoS类别

#### 应用类别的丢包率要求

| 应用类型 | 丢包率要求 | 说明 |
|---------|-----------|------|
| **会话语音** | <1% (10^-2) | 可接受 |
| **会话语音** | <3% (3×10^-2) | 勉强可接受 |
| **流媒体音频** | <1% (10^-2) | 良好 |
| **流媒体视频** | <1% (10^-2) | 良好 |
| **交互数据** | <0.1% (10^-3) | 优秀 |

### 1.3 IETF RFC 3393（时延抖动和丢包）

**标准名称**：RFC 3393 "IP Packet Delay Variation Metric for IP Performance Metrics (IPPM)"

**官方下载**：https://www.rfc-editor.org/rfc/rfc3393

**免费获取**：完全公开，无需注册

### 1.4 军事无线通信的丢包率范围（综合）

基于以上公开标准，军事无线通信的丢包率评估范围：

| 等级 | 丢包率范围 | 评分 | 适用场景 |
|------|-----------|------|---------|
| **优秀** | <10^-4 (0.01%) | 90-100 | 关键数据传输 |
| **良好** | 10^-4 ~ 10^-3 (0.01%-0.1%) | 80-89 | 战术数据链 |
| **中等** | 10^-3 ~ 10^-2 (0.1%-1%) | 70-79 | 语音通信 |
| **及格** | 10^-2 ~ 3×10^-2 (1%-3%) | 60-69 | 短波恶劣条件 |
| **较差** | 3×10^-2 ~ 5×10^-2 (3%-5%) | 40-59 | 极端环境 |
| **不合格** | >5×10^-2 (>5%) | <40 | 无法正常通信 |

**数据来源**：
- ITU-T Y.1541（主要）
- ITU-T G.1010（补充）
- 短波通信实践经验（3%-5%为恶劣电离层条件）

## 三、频谱效率（PO_spectral_efficiency）

### 3.1 ITU-R M.1645 标准（主要参考）

**标准名称**：ITU-R M.1645 "Framework and overall objectives of the future development of IMT-2000 and systems beyond"

**官方下载**：https://www.itu.int/rec/R-REC-M.1645/

**标准内容**：定义了频谱效率的评估框架

#### 频谱效率定义

```
频谱效率 (bps/Hz) = 数据速率 (bps) / 信道带宽 (Hz)
```

| **HSPA** | 1.0-2.0 | ITU-R M.1645 |

## 五、如何使用这些数据进行效能打分

### 5.1 丢包率打分公式

```python
def score_packet_loss_ratio(plr):
    """
    丢包率打分函数
    输入：plr (0-1之间的小数，如0.01表示1%)
    输出：分数 (0-100)
    """
    if plr < 0.0001:  # <0.01%
        return 100
    elif plr < 0.001:  # 0.01%-0.1%
        return 90 + (0.001 - plr) / 0.0009 * 10
    elif plr < 0.01:  # 0.1%-1%
        return 80 + (0.01 - plr) / 0.009 * 10
    elif plr < 0.03:  # 1%-3%
        return 60 + (0.03 - plr) / 0.02 * 20
    elif plr < 0.05:  # 3%-5%
        return 40 + (0.05 - plr) / 0.02 * 20
    else:  # >5%
        return max(0, 40 - (plr - 0.05) * 400)
```

### 5.2 吞吐量打分公式（分系统类型）

#### 短波系统

```python
def score_throughput_hf(throughput_kbps):
    """
    短波吞吐量打分
    输入：throughput_kbps (kbps)
    输出：分数 (0-100)
    """
    if throughput_kbps >= 9.6:
        return 100
    elif throughput_kbps >= 4.8:
        return 80 + (throughput_kbps - 4.8) / 4.8 * 20
    elif throughput_kbps >= 2.4:
        return 70 + (throughput_kbps - 2.4) / 2.4 * 10
    elif throughput_kbps >= 1.2:
        return 60 + (throughput_kbps - 1.2) / 1.2 * 10
    else:
        return max(0, throughput_kbps / 1.2 * 60)
```

#### 超短波系统

```python
def score_throughput_vhf(throughput_mbps):
    """
    超短波吞吐量打分
    输入：throughput_mbps (Mbps)
    输出：分数 (0-100)
    """
    if throughput_mbps >= 2.0:
        return 100
    elif throughput_mbps >= 1.0:
        return 80 + (throughput_mbps - 1.0) * 20
    elif throughput_mbps >= 0.256:
        return 70 + (throughput_mbps - 0.256) / 0.744 * 10
    elif throughput_mbps >= 0.064:
        return 60 + (throughput_mbps - 0.064) / 0.192 * 10
    else:
        return max(0, throughput_mbps / 0.064 * 60)
```

### 5.3 频谱效率打分公式

```python
def score_spectral_efficiency(efficiency_bps_hz):
    """
    频谱效率打分
    输入：efficiency_bps_hz (bps/Hz)
    输出：分数 (0-100)
    """
    if efficiency_bps_hz >= 4.0:
        return 100
    elif efficiency_bps_hz >= 3.0:
        return 80 + (efficiency_bps_hz - 3.0) * 20
    elif efficiency_bps_hz >= 2.0:
        return 70 + (efficiency_bps_hz - 2.0) * 10
    elif efficiency_bps_hz >= 1.0:
        return 60 + (efficiency_bps_hz - 1.0) * 10
    elif efficiency_bps_hz >= 0.5:
        return 50 + (efficiency_bps_hz - 0.5) * 20
    elif efficiency_bps_hz >= 0.3:
        return 40 + (efficiency_bps_hz - 0.3) * 50
    else:
        return max(0, efficiency_bps_hz / 0.3 * 40)
```

## 八、常见问题（FAQ）

### Q1: 为什么不同标准的数值范围不同？
**A**: 不同标准针对不同应用场景：
- ITU-T Y.1541 针对IP网络（通用）
- ETSI/TIA 针对专业移动无线电（PMR）
- ITU-R M.1645 针对移动通信系统（IMT）

建议：根据实际系统类型选择对应标准。

### Q2: 如何选择合适的评估标准？
**A**: 
- 短波系统 → STANAG 4285
- 超短波系统 → ETSI EN 300 392-2 (TETRA) 或 TIA-102 (P25)
- 卫星系统 → ITU-R M.1457
- 丢包率 → ITU-T Y.1541（通用）
- 频谱效率 → ITU-R M.1645（通用）

### Q3: 这些标准是否适用于军事通信？
**A**: 是的，这些都是公开的国际标准，军事通信系统通常参考或超越这些标准。军事专用标准（如GJB、MIL-STD）通常更严格，但不公开。
